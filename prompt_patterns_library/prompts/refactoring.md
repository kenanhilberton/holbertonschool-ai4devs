# Refactoring and Development Prompt Templates Library

This file contains a collection of reusable prompt templates for various development tasks, structured with roles, tasks, and placeholders.

## 1. Code Refactoring Template
**Role**: Senior Software Architect  
**Task**: Refactor the provided [LANGUAGE] code for better clarity and performance.  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: Clean, optimized code with inline comments explaining the changes.

## 2. Style Enforcement Template
**Role**: Tech Lead  
**Task**: Enforce consistent naming and formatting on the following code according to [STYLE_GUIDE].  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: Reformatted code block following the specified style rules.

## 3. Code Review Template
**Role**: Senior Peer Reviewer  
**Task**: Identify logical flaws, anti-patterns, or performance bottlenecks in this code.  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: List of issues found and actionable suggestions for fixes.

## 4. Root Cause Analysis Template
**Role**: Debugging Specialist  
**Task**: Identify the root cause of the error based on the provided logs and code.  
**Input Placeholder**: [ERROR_LOG] and [CODE_BLOCK]  
**Expected Output**: Detailed explanation of the bug and a suggested fix.

## 5. Security Audit Template
**Role**: Cybersecurity Engineer  
**Task**: Scan the following code for vulnerabilities like SQL Injection or XSS.  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: Vulnerability report and a secured version of the code.

## 6. Docstring Generation Template
**Role**: Technical Writer  
**Task**: Generate professional documentation (docstrings) for the following function in [LANGUAGE].  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: Original code updated with professional docstrings in [FORMAT] style.

## 7. README Authoring Template
**Role**: Project Lead  
**Task**: Create a comprehensive README.md file based on project metadata.  
**Input Placeholder**: [PROJECT_DETAILS]  
**Expected Output**: Well-structured Markdown file with project overview and setup steps.

## 8. Unit Test Generation Template
**Role**: QA Automation Engineer  
**Task**: Create a comprehensive unit test suite for the following [LANGUAGE] code.  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: Ready-to-run test file using [TESTING_FRAMEWORK].

## 9. Edge Case Discovery Template
**Role**: Software Tester  
**Task**: Find boundary conditions and edge cases that could break this logic.  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: List of at least 5 edge cases and corresponding test inputs.

## 10. Mock Data Generation Template
**Role**: Database Administrator  
**Task**: Generate realistic mock data based on the provided schema.  
**Input Placeholder**: [DATA_SCHEMA]  
**Expected Output**: Array of mock data objects in JSON or SQL format.

## 11. Performance Optimization Template
**Role**: Performance Engineer  
**Task**: Identify execution bottlenecks and suggest optimizations for time/space complexity.  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: Complexity analysis and an optimized implementation.

## 12. Technical Tutorial Template
**Role**: Developer Advocate  
**Task**: Explain how the following complex code works to a junior developer.  
**Input Placeholder**: [CODE_BLOCK]  
**Expected Output**: A step-by-step technical guide with analogies.
