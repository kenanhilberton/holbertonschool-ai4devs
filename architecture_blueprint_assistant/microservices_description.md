# Microservices Architecture Description

The EcoRide Connect platform is divided into 8 distinct microservices, each with a specific business responsibility and independent data store.

### Core Services:

1. **Authentication Service**: Manages secure identity verification, password encryption, and JWT token issuance.
2. **Passenger Profile Service**: Stores passenger-specific data, including trip history, payment methods, and user preferences.
3. **Driver Verification Service**: A compliance-focused service that validates driver licenses, vehicle registration, and background checks.
4. **Ride Matching Engine**: Uses geospatial data to pair active drivers with nearby passengers based on optimized route efficiency.
5. **Financial Transaction Service**: Handles secure payments, processes refunds, and manages the digital wallet infrastructure for fare splitting.
6. **Real-time Notification Service**: An asynchronous service that triggers push notifications and SMS updates for ride status changes.
7. **Carbon Footprint Tracker**: This service monitors environmental impact by calculating $CO_{2}$ emissions saved per kilometer, fuel efficiency per vehicle type, and total carbon offsets generated by carpooling.
8. **Business Intelligence Service**: Aggregates system-wide data to generate performance reports, heatmaps, and financial forecasts.
